## 1. はじめに

### 1.1 このガイドラインの目的

本ガイドラインは、故・伊藤陽介さん（1988.11.17 – 2026.04.03）の追悼サイト「yosuke-kiddy」のデザインに関する統一基準を定めるものである。

対象読者はデザイナーおよびフロントエンドエンジニア。本書に従うことで、サイト全体の視覚的一貫性と、追悼の場にふさわしい品格を保つ。

### 1.2 ブランドコンセプト

**「モダンミニマル × 陽介さんらしさ」**

白い余白が生む静けさの中に、故人が愛したマゼンタピンクと市松模様をさりげなく添える。派手さではなく「品格のある温かさ」で、訪問者の心に寄り添うデザインを目指す。

### 1.3 プラットフォーム

- **モバイルのみ対応**（スマートフォンからのアクセスが大半）
- フレームワーク: Next.js (App Router) + Tailwind CSS
- ホスティング: Vercel

## 2. デザイン原則

本プロジェクトにおけるデザイン判断は、以下の5つの原則に基づく。

### 原則1: 静謐な余白（Quiet Space）

> 余白は「何もない」ではなく「静けさ」である。

- セクション間の余白を十分に確保し、視覚的な圧迫感を排除する
- 情報を詰め込まず、1セクション1メッセージを徹底する
- **適用例**: セクション間の余白は80〜120px。ヒーロー部はテキスト中心で、写真は透明度を下げた背景として控えめに配置する

### 原則2: 温かみのある格式（Warm Formality）

> 儀礼の品格と、故人の人柄が同居する。

- 儀礼的な情報（式の案内、フォーム）にはヒラギノ明朝の格式を用い、個人的な想い（ヒーローコピー、メッセージ）にはKleeの温かみを用いる
- **適用例**: セクション見出しはヒラギノ明朝W6、ヒーローの縦書きコピーはKlee One SemiBold

### 原則3: さりげない「らしさ」（Subtle Identity）

> 故人のアイデンティティは主張ではなく、余韻として漂わせる。

- マゼンタピンク（`#C2185B`）や市松模様は装飾の主役にしない。ディバイダーや背景パターンとして極めて控えめに使う
- **適用例**: 市松模様はopacity 0.03〜0.05で背景に敷く。アクセントカラーはCTAボタンとセクション区切り線のみに使用

### 原則4: 迷わせない導線（Clear Path）

> 悲しみの中にいる人に「考えさせない」。

- フォームの条件分岐は自動化し、必要な入力だけを表示する
- ボタンやリンクは動詞で具体的に（「送信する」「地図を開く」）
- **適用例**: 住所フィールドは参列系を選択した場合のみ表示。郵便番号から住所を自動補完

### 原則5: アクセシビリティは前提（Accessibility First）

> すべての人が故人を偲べるように。

- コントラスト比4.5:1以上を必ず確保する
- 色だけに頼らず、テキスト・形状・アイコンを併用する
- **適用例**: エラー表示は赤色＋テキスト説明＋アイコンの3要素で構成

## 3. ブランド要素

### 3.1 ロゴ

- **形状**: 正方形（制作予定。完成まではダミー枠で対応）
- **最小表示サイズ**: 32×32px
- **余白規定**: ロゴの1辺の25%以上をクリアスペースとして確保
- **配置**: フッターまたはヘッダーに小さく配置。ヒーロー部には使用しない

### 3.2 トーン＆マナー

**基本トーン: 温かみのある柔らかい敬語**

サイト全体を通じて、丁寧でありながら冷たくない、温もりのある文体を使う。

| 場面             | トーン           | 例                                                           |
| ---------------- | ---------------- | ------------------------------------------------------------ |
| 式の案内         | 丁寧な敬語       | 「ご参列をお待ちしております」                               |
| 喪主からの紹介文 | 柔らかい語り口   | 「陽介は、いつも周りを明るくしてくれる人でした」             |
| フォーム案内     | 親しみやすい敬語 | 「陽介に伝えたいことがあれば、ご記入ください」               |
| 完了メッセージ   | 温かい感謝       | 「ご記帳ありがとうございます。陽介もきっと喜んでいます。」   |
| ヒーローコピー   | 詩的・内省的     | 「『信じられない』を受け止めて、私たちは明日も生きていく。」 |

**推奨ワード**: 「偲ぶ」「伝える」「想い」「ありがとう」

**避けるべき表現**: 事務的すぎる言い回し（「処理が完了しました」）、カジュアルすぎるスラング、死を直接的に表現する強い言葉

### 5.3 縦書き（writing-mode）

ヒーローセクションのコピーは `writing-mode: vertical-rl` で縦書き表示する。

```css
.hero-copy {
  writing-mode: vertical-rl;
  text-orientation: mixed;
  font-family: "Klee One", serif;
  font-weight: 600;
  font-size: 1.5rem;
  line-height: 2;
  letter-spacing: 0.1em;
  color: #1a1a1a;
}
```

- 縦書きテキストは画面中央〜やや右寄りに配置
- 背景写真は透明度を下げて（opacity: 0.15〜0.25）テキストの可読性を確保
- 句読点の位置や改行は手動で調整し、自然な読み心地を保つ（Must）

## 7. UIコンポーネント仕様

### 7.1 ボタン（petit-pas風 × ボーダー描画アニメーション）

参考: petit-pasのミニマルなゴーストボタンを踏襲。シャープな矩形＋細ボーダー＋右三角アイコン。

#### 3タイプの定義

| 種類 | 通常状態 | タップ時 | 用途 |
| --- | --- | --- | --- |
| **プライマリ** | 背景 `#a06a8c` opacity 0.15 / ボーダー 1px `#a06a8c` / テキスト `#484848` | 背景 `#a06a8c` 塗り / テキスト `#FFFFFF` / ボーダー描画アニメーション | フォーム送信「送信する」等のCTA |
| **セカンダリ** | 背景 transparent / ボーダー 1px `#484848` / テキスト `#484848` | 背景 `#484848` opacity 0.08 / ボーダー描画アニメーション | 「地図を開く」「CONCEPT」等 |
| **ゴースト** | 背景 transparent / ボーダーなし / テキスト `#636363` / 下線1px | テキスト `#484848` / 下線2px | キャンセル等の控えめなアクション |

#### アイコン

- ボタン右端に **右三角形 `▶`** を配置（petit-pas準拠）
- サイズ: 10×10px
- 色: テキスト色と同色
- テキストとアイコンの間隔: 12px
- ゴーストボタンにはアイコンなし

#### ボーダー描画アニメーション（全ボタン共通）

タップ時にボーダーが左上から時計回りに描画されるアニメーション。

```
【通常状態】
  送信する   ▶        ← ボーダーは opacity 0.3 で薄く存在
                        （ボタンの存在は見えるが主張しない）

【タップ時：ボーダー描画】
┌────────────────┐     ← 左上から時計回りにボーダーが描画
│  送信する   ▶  │       stroke-dashoffset: 100% → 0%
└────────────────┘       duration: 400ms / ease-in-out
                         同時に背景色がふわっと変化
```

**実装方針**:
- SVG `<rect>` で `stroke-dasharray` + `stroke-dashoffset` アニメーション
- `touchstart` でトリガー、`touchend` で逆再生（元に戻る）
- ゴーストボタンは例外: 下線の太さ変化（1px→2px / 200ms）
- `prefers-reduced-motion: reduce` 時はアニメーション無効、即時状態変更のみ（Must）

#### 共通スタイル

```
角丸: 2px（シャープミニマル）
高さ: 48px（タッチターゲット確保）
横幅: auto + padding 28px 左右 / 100%（フォーム内）
フォント（英字）: Cormorant Garamond 400 14px / letter-spacing: 0.12em / uppercase
フォント（日本語）: Hiragino Mincho ProN W3 14px / letter-spacing: 0.05em
disabled: opacity 0.35 + cursor not-allowed + アニメーション無効
フォーカスリング: `#a06a8c` の2pxアウトライン + 2pxオフセット
```

## 8. アクセシビリティ

### 8.1 コントラスト基準

- テキスト × 背景: **4.5:1以上**（Must / WCAG 2.1 AA）
- 非テキスト要素（アイコン、ボーダー）× 背景: **3:1以上**（Must）
- 大テキスト（18px以上 bold、または 24px以上）: **3:1以上**

### 8.2 タッチターゲット

- 最小サイズ: **44×44px**（Must）
- ボタン、チェックボックス、リンクのすべてに適用
- 隣接するタッチターゲット間の余白: 8px以上

### 8.3 スクリーンリーダー対応

- 画像にはすべて意味のあるalt属性を付与（Must）
  - 故人の写真: `alt="伊藤陽介さんの写真"`（具体的に）
  - 装飾画像: `alt=""`（空文字）+ `aria-hidden="true"`
- ボタンは具体的な動詞で記述: ✅「送信する」 ❌「こちら」
- フォームのラベルは `<label>` で関連付け（Must）
- エラーメッセージは `aria-live="polite"` で通知

### 8.4 色だけに頼らない情報伝達

- エラー: 赤色 + ⚠アイコン + テキスト説明
- 必須項目: 「\*」マーク + 「必須」テキスト
- チェック状態: 色変更 + ✓アイコン

- **サイズ**: 24×24px（標準）、20×20px（インライン）
- **ライブラリ**: Lucide Icons（推奨）またはSVGカスタム

### 9.2 写真の扱い

| 場所               | 処理                                                |
| ------------------ | --------------------------------------------------- |
| ヒーロー背景       | 全幅表示 / opacity 0.15〜0.25 / `object-fit: cover` |
| About写真          | petit-pas風コラージュ配置 / 角丸0px（エッジを効かせる） / 2〜3枚 |
| 管理画面サムネイル | 正方形クロップ / 80×80px / 角丸2px                  |
| プレースホルダー   | ライトグレー（`#F5F5F5`）にカメラアイコン           |

- すべての写真は `next/image` で最適化する（WebP変換、レスポンシブ）（Must）
- アップロード写真はクライアントサイドで長辺2000pxにリサイズしてから送信

## 10. システム状態とフィードバック

### 10.1 ローディング状態

- フォーム送信中: ボタンをdisabled化 + スピナー表示 + テキスト「送信しています...」
- 写真アップロード中: プログレスバー（`#C2185B`）+ 「アップロード中... (2/5)」

## 11. アニメーション

### 11.1 方針

追悼サイトの品格を保つため、アニメーションは最小限に抑える。

### 11.2 許可されるアニメーション

| 種類           | 仕様                                               | 用途                           |
| -------------- | -------------------------------------------------- | ------------------------------ |
| フェードイン   | opacity 0→1 / duration: 600ms / ease-out | セクションのスクロール連動表示 |
| スライドアップ | translateY(20px→0) + opacity 0→1 / duration: 600ms | テキスト要素の出現 |
| ボタン ボーダー描画 | stroke-dashoffset 100%→0% / duration: 400ms / ease-in-out / 左上から時計回り | ボタンタップ時のフィードバック（touchend で逆再生） |
| ボタン背景フィル | opacity 0→1（プライマリ）or 0→0.08（セカンダリ）/ duration: 300ms | ボーダー描画と同時に背景が塗られる |

### 11.3 禁止されるアニメーション

- バウンス、スプリング系の弾むアニメーション
- パララックススクロール
- 自動再生のスライドショー
- 3秒を超える長いアニメーション
- `prefers-reduced-motion: reduce` 設定時はすべてのアニメーションを無効化する（Must）

## 12. Must（必須）と Should（推奨）のまとめ

### Must（必須） — 必ず守ること

| #   | ルール                                                            |
| --- | ----------------------------------------------------------------- |
| M1  | テキストと背景のコントラスト比は4.5:1以上を確保する               |
| M2  | タッチターゲットは最小44×44pxを確保する                           |
| M3  | 色だけで情報を伝えない。テキスト・アイコンを必ず併用する          |
| M4  | すべての画像にalt属性を付与する                                   |
| M5  | フォームラベルは `<label>` で関連付ける                           |
| M6  | エラーメッセージは3要素（何が/なぜ/どうする）で構成する           |
| M7  | 写真は `next/image` で最適化する                                  |
| M8  | 市松模様の使用はページ内2〜3箇所まで                              |
| M9  | 縦書きコピーの改行位置は手動で調整する                            |
| M10 | `prefers-reduced-motion: reduce` 時にアニメーションを無効化する   |
| M11 | ライトモードのみ。ダークモード非対応                              |
| M12 | フォントはセルフホスト。CDNフォールバックに依存しない（英字除く） |

### Should（推奨） — 状況に応じて柔軟に対応

| #   | ルール                                                                            |
| --- | --------------------------------------------------------------------------------- |
| S1  | セクション間の余白は80〜120pxの範囲で調整する                                     |
| S2  | ヒーロー背景写真のopacityは0.15〜0.25の範囲で、テキストの可読性を見ながら調整する |
| S3  | フォーム送信完了のトーストは5秒で自動消去する                                     |
| S4  | About写真はpetit-pas風のコラージュ配置にする                                      |
| S5  | アイコンはLucide Iconsを使用する                                                  |
| S6  | スクロール連動アニメーションのdurationは600msを基準とする                         |
| S7  | ロゴ完成後、ダミー枠をロゴ画像に差し替える                                        |
